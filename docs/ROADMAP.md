# Wave Router Product Roadmap

## Phase 1 — Functional Hackathon Demo

- Standalone dashboard served by the Node application
- Provider capability registry
- Constraint-based route quotes
- Job execution lifecycle
- Balance reservation and posting
- Visible ILP and XRPL settlement stages
- Recent job history

## Phase 2 — Real AI Routing

- Provider adapter interface
- OpenAI-compatible provider adapter
- Local model adapter
- Health checks and live capacity
- Token and latency metering
- Output verification and fallback routing

## Phase 3 — Financial Integrity

- TigerBeetle account and transfer model
- Pending transfers for job reservations
- Idempotent posting and voiding
- Provider payables and platform fees
- Reconciliation reports

## Phase 4 — XRPL Settlement

- Testnet deposit wallet
- Deposit indexing and account crediting
- Provider payout transactions
- XRPL pathfinding for cross-asset settlement
- Transaction hashes and explorer links

## Phase 5 — ILP / Open Payments

- Rafiki or compatible wallet integration
- Payment quotes
- Outgoing payments
- Streaming payment checkpoints
- Reconciliation against TigerBeetle

## Phase 6 — Open Provider Network

- Signed provider manifests
- Benchmark-based reputation
- Privacy and jurisdiction policies
- Provider collateral and disputes
- Multi-step intelligence routes

The project will avoid speculative token issuance until routing, accounting, and settlement work reliably.
